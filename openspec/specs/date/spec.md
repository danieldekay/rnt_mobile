## Purpose

The Date Utilities domain provides helper functions for formatting, parsing, and manipulating dates throughout the RNT Mobile application. It ensures consistent date presentation in German locale and provides utilities for API communication and filtering.

## Requirements

### Requirement: Date Formatting for Display
The system SHALL provide functions to format dates for user display in German locale.

#### Scenario: Format full date
- **GIVEN** a valid date string or Date object
- **WHEN** `formatDate` is called
- **THEN** the date is formatted as "d. MMMM yyyy" (e.g., "9. Mai 2026")
- **AND** the German locale is used for month and day names

#### Scenario: Format short date
- **GIVEN** a valid date string or Date object
- **WHEN** `formatShortDate` is called
- **THEN** the date is formatted as "d. MMM" (e.g., "9. Mai")
- **AND** the German locale is used for month names

#### Scenario: Format day and month with weekday
- **GIVEN** a valid date string or Date object
- **WHEN** `formatDayAndMonth` is called
- **THEN** the date is formatted as "EEE, d. MMM" (e.g., "Sa, 9. Mai")
- **AND** the German locale is used for weekday and month names

#### Scenario: Handle invalid dates gracefully
- **GIVEN** an invalid date, null, or undefined
- **WHEN** any date formatting function is called
- **THEN** an empty string is returned
- **AND** no errors are thrown

### Requirement: Date Formatting for API Communication
The system SHALL provide functions to format dates for API requests.

#### Scenario: Format date for API
- **GIVEN** a valid date string or Date object
- **WHEN** `formatApiDate` is called
- **THEN** the date is formatted as "yyyy-MM-dd" (e.g., "2026-05-09")
- **AND** the format matches the Tribe Events API expectations
- **AND** invalid dates return an empty string

#### Scenario: Format time for display
- **GIVEN** a valid date string or Date object
- **WHEN** `formatTime` is called
- **THEN** the time is formatted as "HH:mm" (e.g., "19:30")
- **AND** invalid dates return an empty string

#### Scenario: Format time range
- **GIVEN** two valid date strings or Date objects representing start and end times
- **WHEN** `formatTimeRange` is called
- **THEN** if times are different, they are formatted as "startTime – endTime" (e.g., "19:00 – 22:00")
- **AND** if times are the same, only one time is returned
- **AND** if start time is invalid, an empty string is returned
- **AND** the separator uses an en-dash character with proper spacing

### Requirement: Date Filtering Utilities
The system SHALL provide utilities for working with date filters and ranges.

#### Scenario: Get date range for filter
- **GIVEN** a date filter string ("today", "week", "month", "all")
- **WHEN** `getDateRange` is called in the event store
- **THEN** appropriate start and end dates are returned based on the filter
- **AND** "today" returns the current day (00:00 to 23:59)
- **AND** "week" returns current day through next 7 days
- **AND** "month" returns current month through last day of month
- **AND** "all" returns current date through 3 years in future

#### Scenario: Get month date range
- **GIVEN** a specific Date object
- **WHEN** `getMonthDateRange` is called in the event store
- **THEN** the first and last day of the month are returned
- **AND** times are set to 00:00:00.000 and 23:59:59.999 respectively
- **AND** the function works correctly for all months including February

#### Scenario: Get month key
- **GIVEN** a specific Date object
- **WHEN** `getMonthKey` is called in the event store
- **THEN** a string in "YYYY-MM" format is returned
- **AND** the key is used for caching monthly calendar data
- **AND** leading zeros are used for month values (e.g., "2026-05")
