## Purpose

The API domain handles all interactions with external services, primarily the Rhein-Neckar-Tango WordPress Events API. It provides functions for fetching events, venues, organizers, and specialized data like DJ information, along with data normalization utilities.

## Requirements

### Requirement: Fetch Events from Tribe API
The system SHALL provide functions to fetch event data from the Tribe Events API with proper normalization.

#### Scenario: Fetch events with filtering
- **GIVEN** valid parameters for event types, music type, and date filter
- **WHEN** `fetchEvents` is called
- **THEN** a request is made to the Tribe Events API with appropriate query parameters
- **AND** the response is normalized to clean HTML entities and format data

#### Scenario: Fetch all events with pagination limits
- **GIVEN** the API may return many pages of results
- **WHEN** `fetchAllEvents` is called
- **THEN** pagination is limited to a maximum of 10 pages
- **AND** a 2-minute timeout prevents infinite requests
- **AND** partial results are returned gracefully if limits are reached

#### Scenario: Fetch event by ID
- **GIVEN** a valid event ID
- **WHEN** `fetchEventById` is called
- **THEN** a request is made to the specific event endpoint
- **AND** the response is normalized
- **AND** errors are wrapped in `EventFetchError` with status code

### Requirement: Fetch Venues and Organizers
The system SHALL provide functions to fetch venue and organizer data from the API.

#### Scenario: Fetch venues
- **GIVEN** the venues API endpoint
- **WHEN** `fetchVenues` is called
- **THEN** venue data is fetched and normalized
- **AND** only venues with valid IDs and names are returned

#### Scenario: Fetch organizers with deduplication
- **GIVEN** the organizers API endpoint with pagination
- **WHEN** `fetchOrganizers` is called
- **THEN** organizer data is fetched across multiple pages
- **AND** duplicate organizers are filtered out by ID
- **AND** only organizers with valid IDs and names are returned

### Requirement: Data Normalization
The system SHALL provide utility functions to normalize and clean data from the API.

#### Scenario: Decode HTML entities
- **GIVEN** text containing HTML entities
- **WHEN** `decodeHtmlEntities` is called
- **THEN** entities like `&amp;`, `&lt;`, `&gt;` are converted to their character equivalents
- **AND** numeric entities are handled correctly

#### Scenario: Normalize text
- **GIVEN** text with extra whitespace or null values
- **WHEN** `normalizeText` is called
- **THEN** HTML entities are decoded, non-breaking spaces are converted to regular spaces
- **AND** multiple consecutive spaces are reduced to single spaces
- **AND** leading/trailing whitespace is trimmed

#### Scenario: Normalize event data
- **GIVEN** a raw Tribe event object
- **WHEN** `normalizeEvent` is called
- **THEN** the event's description is processed to replace obfuscated email markup
- **AND** text fields are normalized
- **AND** venue and organizer data are normalized if present

#### Scenario: Format event cost
- **GIVEN** a cost string in various formats
- **WHEN** `formatEventCost` is called
- **THEN** free events are identified and return "Frei"
- **AND** Euro amounts are formatted with comma as decimal separator
- **AND** costs with Euro markers are properly formatted
- **AND** unrecognized costs are returned as normalized text

### Requirement: Specialized Data Extraction
The system SHALL provide functions to extract specific information from event data.

#### Scenario: Extract DJ name from description
- **GIVEN** an event description containing DJ information
- **WHEN** `extractDjFromDescription` is called
- **THEN** text matching DJ patterns is processed
- **AND** plausible DJ names are identified and returned
- **AND** null is returned if no valid DJ name is found

#### Scenario: Extract workshop information from description
- **GIVEN** an event description containing workshop information
- **WHEN** `extractWorkshopFromDescription` is called
- **THEN** text following "Workshop" or "Workshops" is extracted
- **AND** the extracted text is normalized
- **AND** null is returned if no workshop information is found
