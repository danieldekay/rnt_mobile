## Purpose

The RNT Mobile application is a Progressive Web App (PWA) built with SvelteKit 5 and Svelte 5 that displays Rhein-Neckar-Tango events fetched from the WordPress Events API. It provides event listings with filtering capabilities, calendar views, and detailed event information.

## Requirements

### Requirement: Event Data Fetching
The system SHALL fetch event data from the Rhein-Neckar-Tango WordPress Events API and normalize it for display.

#### Scenario: Fetch events from Tribe API
- **GIVEN** the application is initialized
- **WHEN** the event store loads events
- **THEN** data is fetched from `https://www.rhein-neckar-tango.de/wp-json/tribe/events/v1`
- **AND** the response is normalized to remove HTML entities and format dates/costs

#### Scenario: Handle API pagination limits
- **GIVEN** the API returns more than 10 pages of results
- **WHEN** `fetchAllEvents` is called
- **THEN** only 10 pages are fetched regardless of `total_pages`
- **AND** the function returns collected events without error

#### Scenario: Handle API timeouts
- **GIVEN** API requests exceed 2 minutes
- **WHEN** fetching events
- **THEN** requests are aborted via AbortController
- **AND** collected events are returned without throwing AbortError to caller

#### Scenario: Filter by event type

- **THEN** only events matching the selected types are displayed

#### Scenario: Filter by music type
- **GIVEN** events are loaded in the store
- **WHEN** user toggles a music filter (traditional, mixed, neo)
- **THEN** only events with matching music categories are displayed
- **AND** filter state is maintained in the event store

#### Scenario: Filter by date range
- **GIVEN** events are loaded in the store
- **WHEN** user selects a date filter (today, week, month, all)

- **AND** filter state is maintained in the event store
